19 JMS 리소스

JMS리소스에는 컨넥션 팩토리, 토픽/큐 데스터네이션, 클라이언트가 있다.

참고 : JMS 리소스를 사용하기 위해서는 JMS 엔진이 엔진 컨테이너에 포함되어야 한다.

19.1 JMS 리소스 구성

컨넥션 팩토리는클라이언트 어플리케이션 사용을 편리하게 하기 위해서 JEUS JMS 서버에서 연결을 만들고 저장하는데 사용이 된다.

데스터네이션은 JMS 클라이언트 어플리케이션들이 메시지들을 보내고 받는 장소이다.

19.1.1 컨넥션 팩토리 추가

JEUS 매니저 > 엔진 컨테이너 개요 > 엔진 컨테이너 > JMS 리소스 > 새 컨넥션 팩토리 생성

1. 노드 트리에서 컨넥션 팩토리 폴더를 클릭한 후 새 컨넥션 팩토리 생성 링크를 클릭하거나 컨텍스트 메뉴에서 새 컨넥션 팩토리 생성 을선택한다.

그림 166 새 컨넥션 팩토리 생성 선택

2. 기본 설정을 입력한다. 여기서는 컨텍션 팩토리 타입을 설정한다. 타입에는 총 6가지(nonxa, xa, queue, topic, xaqueue, xatopic)가있다.

그림 167 새 컨넥션 팩토리 생성 - 1단계 기본 설정

3. 쓰레드 풀 설정을 한다.

그림 168 새 컨넥션 팩토리 생성 -2단계 쓰레드 풀 설정

4. 추가할 컨넥션 팩토리를 생성과 동시에 바인딩 하고 싶다면 바인딩 선택 항목을 선택한 후 생성 버튼을 클릭한다.

그림 169 새 컨넥션 팩토리 생성 -3단계 생성

5. 컨넥션 팩토리가 추가되면 노드 트리의 컨넥션 팩토리 폴더에 컨넥션 팩토리의 아이콘( )이 나타난다. .

JEUS 매니저 > 엔진 컨테이너 개요 > 엔진 컨테이너 > JMS 리소스 > 새토픽 데스터네이션 생성

1. 노드 트리에서 데스터네이션 폴더를 클릭한 후 새 토픽 데스터네이션 생성 링크를 클릭하거나 컨텍스트 메뉴에서 새 토픽 데스터네이션 생성 을선택한다.

2. 기본 설정을 입력한다.

그림 170 새 토픽 데스터네이션 생성 -1단계 기본 설정

3. 추가할 토픽 데스터네이션 생성과 동시에 바인딩 하고 싶다면 바인딩 선택 항목을 선택한 후 생성 버튼을 클릭한다.

그림 171 토픽 데스터네이션 생성 - 2단계 생성

4. 토픽 데스터네이션이추가되면 노드 트리의 데스터네이션 폴더에 토픽 데스터네이션 아이콘( 또는 )이 나타난다.

19.1.3 큐 데스터네이션 추가

JEUS 매니저 > 엔진 컨테이너 개요 > 엔진 컨테이너 > JMS 리소스 > 새 큐 데스터네이션 생성

그림173 큐 데스터네이션 생성 - 2단계 생성

19.2 JMS 리소스 설정

19.2.1 컨넥션 팩토리기본 설정

JEUS 매니저 > 엔진 컨테이너 개요 > 엔진 컨테이너 > JMS 리소스 > 설정 > 컨넥션 팩토리 기본 설정

1. 노드 트리에서 변경하고자 하는 컨넥션 팩토리 를클릭한다.

4. 입력이 완료되면 확인 버튼을 클릭한다.

19.3 JMS 리소스 제어

19.3.1 컨넥션팩토리 바인드

JEUS 매니저 > 엔진 컨테이너 개요 > 엔진 컨테이너 > JMS 리소스 > 제어 > 컨넥션 팩토리 바인드

1. 노드 트리에서 JMS의 컨넥션 팩토리 폴더를 클릭한다.

3. 컨넥션 팩토리목록에서 해당 컨넥션 팩토리의 바인드 버튼이 비활성화 되며 노드트리에는 활성화된 아이콘( )이 나타난다.

19.3.3 듀러블 서브스크라이버(Durable Subscriber) 에서 메시지 삭제

JEUS 매니저 > 엔진 컨테이너 개요 > 엔진 컨테이너 > JMS 리소스 > 토픽데스터네이션 > 듀러블 서브스크라이버 > 메시지 삭제

듀러블 서브스크라이버 메시지의 경우 각각의 듀러블 서브스크라이버의 메시들 정보(생성시간, 전송여부, 타입)에따라 관리자가 삭제할 수가 있다.

1. 노드 트리에서 메시지를 삭제하고자 하는 듀러블 서브스크라이버 를가지는 토픽 데스터네이션 을클릭한다.

2. 하위 탭에서 듀러블 서브스크라이버 를선택한다.

3. 듀러블 서브스크라이버 목록에서 메시지를 삭제하고자 하는 듀러블 서브스크라이버를 선택한다.

4. 메시지 탭을 클릭한다.

5. 메시지 정보를 보고 삭제할 메시지를 선택한다.

6. 삭제 버튼을 클릭한다.

19.4 JMS 리소스 통계

19.4.1 듀러블 서브스크라이버 통계

JEUS 매니저 > 엔진 컨테이너 개요 > 엔진 컨테이너 > JMS 리소스 > 토픽데스터네이션 > 듀러블 서브스크라이버 > 통계

듀러블 서브스크립숀은 클라이언트에 메시지를 발송하는 것을 보장하는 토픽 데스터네이션에서 사용된다. 듀러블 서브스크립숀은 수신자가 연결을 종결할때 때, 비활성화된 것으로 간주하여 클라이언트가 비활성화된 동안 메시지를 저장해 두었다가, 활성화 상태일 때 듀러블 서브스크라이버에 메시지를 발송 한다.

듀러블 서브스크라이버는 토픽 데스터네이션일 경우만 사용된다.

1. 노드 트리에서 모니터링 하고자 하는 토픽 데스터네이션 을클릭한다.

2. 듀러블 서브스크라이버 탭을 선택한다.

3. 통계 정보가 출력된다.

19.4.2 소비자(Cusumer) 통계

JEUS 매니저 > 엔진 컨테이너 개요 > 엔진 컨테이너 > JMS 리소스 > 데스터네이션 >소비자 > 통계

데스터네이션으로부터 메시지를 수신하는 클라이언트에 대한 정보를 보여준다.

1. 노드 트리에서 모니터링 하고자 하는 데스터네이션 을클릭한다.

2. 소비자 탭을 클릭한다.

3. 통계 정보가 출력된다.

19.4.3 클라이언트 통계

JEUS 매니저 > 엔진 컨테이너 개요 > 엔진 컨테이너 > JMS 리소스 > 클라이언트 > 통계

현재 JMS 엔진에 등록된 모든 클라이언트에 대한 정보를 볼 수 있다.

2. 통계 정보가 출력된다.

19.4.4 클라이언트 연결 통계

JEUS 매니저 > 엔진 컨테이너 개요 > 엔진 컨테이너 > JMS 리소스 > 클라이언트 > 연결 > 통계

4. 세션목록에서 통계를 보고자하는 세션을 클릭한다.

5. 통계 정보가 출력된다.
