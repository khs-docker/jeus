(5) <jeus-web-dd> <context> <context-path>
Description컨텍스트의 URL path이다. 이 path는 도메인 이름(과 포트 번호) 뒤에 붙는다. 예: 만약에 A라는 컨텍스트의 context path가 “/examples”이라면 “http://www.foo.com/examples/index.html” URL은 A 컨텍스트의 index.html파일을 수행한다. 주의: 만약 이 element가 WEBMain.xml에 설정되어 있지 않으면 이곳에 설정을 해야만 한다.
Value DescriptionURL은 “/”로 시작해야 한다. 해당 경로는 Virtual Host내에서 유일해야 한다.
Value Typetoken
(6) <jeus-web-dd> <context> <docbase>
Description이 element는 Web application의 모든 파일들이 저장되는 Context의 도큐먼트 베이스 디렉토리를 설정한다. 이 디렉토리는 Context group의 도큐먼트 베이스 디렉토리에 포함되어 추가된다(context-group><group-docbase>element에 설정된 것처럼). 이는 절대 경로로도 지정이 가능하다. 만일 이 element를 설정하지 않으면 Context는 디폴트 Context디렉토리에 저장된다(Context group디렉토리 밑을 의미한다).
Value Description이 Context에 대해 절대 또는 상대 경로. 만일 상대 경로가 지정되면, 그 경로는 Context group의 도큐먼트 베이스에 더해진다 (디렉토리 분리자 “\”또는 “/”로 시작하지 않는다).
Value Typetoken
